# Experiments

Shared retrieval experimentation phase — all notebooks here apply to both
Project 1 (codebase RAG) and Project 2 (citizen science RAG). This folder
is the foundation both projects build on before their pipelines diverge.

## Notebooks

| Notebook | Investigates |
|---|---|
| `01_naive_to_hybrid` | Chunking strategy, embedding model selection, hybrid retrieval construction |
| `02a_eval_bm25_mangling` | Research artifact — preserves broken whitespace BM25 tokenization to isolate the sparse leg's contribution by comparison with 02b. |
| `02b_eval_baseline` | Authoritative retrieval experiments v1→v3 with identifier-aware BM25 tokenization — eval harness, failure mode diagnosis, LLM summary enrichment against 10 solver Q&A pairs |
| `03_qdrant_migration` | ChromaDB → Qdrant migration, point-by-point verification, hybrid parity confirmation, cross-encoder reranker preview |

## Contents

`qa_pairs/solvers_qa.json` — 10 Q&A pairs covering `kernelpack.solvers`.
Generated by Codex. Three tiers: api, workflow, conceptual. Not yet
domain-expert validated — treat recall numbers as directional.

`metadata/solvers_summaries.json` — plain-English summaries of every function
in `kernelpack.solvers`, generated by Codex. Solvers module only — expansion
to all 6 modules is Phase 0.1 of the research plan.

`scripts/migrate_to_qdrant.py` — standalone migration script. Reads
`chroma_export.pkl`, creates a Qdrant collection, and writes all points.
Run directly or via `03_qdrant_migration.ipynb`.

`utils/chunking.py` — shared chunking utilities including `bm25_tokenize`,
the identifier-aware tokenizer used across 02b and 03. Import from here;
do not re-implement inline.

`utils/eval_utils.py` — shared eval harness (`print_eval_table`, `is_hit`,
recall computation). All notebooks use the same harness so results are
directly comparable.

`reports/` — PDF reports for each experiment series.

| Report | Covers |
|---|---|
| `01_naive_to_hybrid_report.pdf` | Pipeline development sections 1–4: fixed-line → AST chunking → UniXcoder → hybrid RRF. VDB recommendation (ChromaDB → Qdrant). |
| `02_eval_report_v1_pre_fixes.pdf` | Historical context only — pre-BM25 fix, pre-QA correction. Do not treat numbers as ground truth. |
| `solvers_qa_correction.pdf` | QA pair correction: 3 un-indexable targets replaced, before/after recall comparison. |
| `02_eval_report_v2.pdf` | Authoritative eval report: BM25 tokenization fix, v1→v3 recall attribution, wide retrieval diagnostic. |
| `03_qdrant_migration_report.pdf` | Migration verification, hybrid parity, reranker preview. |
