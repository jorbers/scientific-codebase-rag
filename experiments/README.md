# Experiments

Shared retrieval experimentation phase — all notebooks here apply to both
Project 1 (codebase RAG) and Project 2 (citizen science RAG). This folder
is the foundation both projects build on before their pipelines diverge.

## Notebooks

| Notebook | Investigates |
|---|---|
| `01_naive_to_hybrid` | Chunking strategy, embedding model selection, hybrid retrieval construction |
| `02_eval_baseline` | Eval harness and retrieval experiments v1→v3 against 10 solver Q&A pairs |

## Contents

`qa_pairs/solvers_qa.json` — 10 Q&A pairs covering `kernelpack.solvers`.
Generated by Codex, prompted to produce questions that sound like a scientist
who knows what they want numerically but not KernelPack's API. Three tiers:
api, workflow, conceptual. Not yet domain-expert validated — treat recall
numbers as directional.

`metadata/solvers_summaries.json` — plain English summaries of every function
and method in `kernelpack.solvers`, generated by prompting Codex with full
repo context. Prompt used:
```
Here is the repo for a scientific computing library called KernelPack-python.

For each function and method in src/kernelpack/solvers/, write a 1-2 sentence plain English summary of what it does. Use the surrounding class and repo context to inform your description. Do not invent behavior not present in the code.

Return as JSON: [{"function_name": "...", "class_name": "...", "summary": "..."}]
```

Solvers module only — expansion to all 6 modules is Phase 0.1 of the research plan.
